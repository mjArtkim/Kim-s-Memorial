
# [웹사이트 구축 요구사항 정의서] 故 김병후 님 디지털 메모리얼

## 1. 프로젝트 개요

* **목적:** 故 김병후 님의 생애를 기리고, 장지 정보 제공 및 지인들의 추모 공간 마련
* **작업 범위:**
* **웹 디자인 (UI/UX):** 전체 디자인 제안 및 구현
* **프론트엔드 개발:** Vue.js 3 기반 반응형 웹 구현
* **백엔드 연동:** 클라이언트가 제공하는 API 및 데이터 연동


* **대상:** 유가족, 지인, 전 직장 동료 (한국, 중국, 영어권 포함)

## 2. 기술 사양 (Technical Requirements)

* **Frontend:** `Vue.js 3` (Composition API 권장)
* **반응형:** Desktop / Mobile 완벽 대응 필수
* **다국어(i18n):** 한국어, 중국어(간체/번체), 영어 4개 국어 지원 구조 - 최종 텍스트는 목업 디자인 확인 후 제공 예정.


* **Backend Interface:**
* 백엔드(API)는 클라이언트가 직접 개발합니다.
* 프론트엔드는 정의된 **API Endpoint** 규격에 맞춰 데이터를 송수신해야 합니다.


* **Asset Management:**
* 갤러리 미디어는 소스 코드에 하드코딩하지 않고, **서버의 특정 폴더(`/photos`)에 업로드된 파일들을 동적으로 불러오는 구조**여야 합니다.

### D. 갤러리 (Gallery)

* **데이터 소스:**
* API를 통해 `/photos` 폴더 내의 파일명 리스트를 받아와 렌더링합니다.
* 파일명 리스트는 **이름 순(오름차순)**으로 정렬하여 표시합니다.


* **미디어 타입 구분 (Media Type Handling):**
* 파일의 **확장자**를 식별하여 이미지와 영상을 구분해서 렌더링해야 합니다.
* **이미지 (jpg, png 등):** `<img>` 태그로 표시
* **영상 (mp4, webm 등):** `<video controls>` 태그로 표시


* **기능:** Masonry Grid 또는 슬라이드 방식, 썸네일 클릭 시 확대 보기(Lightbox)

## 4. API 인터페이스 (Frontend 연동 규격)

*프론트엔드 작업자는 아래 엔드포인트를 기준으로 개발해 주세요.*

| Method | Endpoint | 설명 | Request | Response Example |
| --- | --- | --- | --- | --- |
| `GET` | `/api/guestbook` | 방명록 목록 조회 | - | `[{ "id": 1, "name": "...", "content": "..." }, ...]` |
| `POST` | `/api/guestbook` | 방명록 작성 | `{ "name": "...", "content": "..." }` | `{ "success": true }` |
| `GET` | `/api/gallery` | 갤러리 파일 목록 조회 | - | `["dad.jpg", "speech.mp4", "award.png"]` |

> **※ 갤러리 구현 팁 (개발자 참고용):**
> * API 응답(`["a.jpg", "b.mp4"]`)을 받아 반복문(`v-for`)을 돌릴 때, **확장자 체크**를 통해 `<video>` 태그와 `<img>` 태그를 분기해 주세요.
> * 소스 경로는 `/photos/파일명` 으로 바인딩하면 됩니다.

## 5. 디자인 및 전달 사항 (Design Guidelines)

1. **디자인 컨셉:** 장례/추모 사이트의 특성을 고려하여 **신뢰감 있고 차분한 톤앤매너**를 유지해 주세요.
2. **텍스트 처리:** `[Temporary]` 문구는 임시 텍스트로 대체하여 디자인해 주세요.
3. **산출물:**
* **Vue.js 소스 코드 원본 (필수):** 추후 텍스트 수정 및 Docker 빌드를 위해 원본 코드가 필요합니다.
* `npm run build` 시 에러 없이 정상적으로 빌드되어야 합니다.
4. **제작자 Credit:** 사이트 최하단(Footer)에 작업자의 아이덴티티를 나타낼 수 있는 로고, 이름, 포트폴리오 링크('Designed & Developed by...') 등을 자유롭게 넣어주세요. 가족이 같이 만드는 공간인 만큼 제작자의 흔적을 남겨주시면 좋겠습니다.
